Step 1: Verify the harness builds cleanly from the command line.

```bash
cd ~/sikai-spike-harness
xcodebuild clean build \
  -project SikAISpike.xcodeproj \
  -scheme SikAISpike \
  -destination "generic/platform=iOS" \
  CODE_SIGNING_ALLOWED=NO \
  2>&1 | tail -20
```

If this reports BUILD SUCCEEDED, proceed. If BUILD FAILED:
  - Read the full error output (xcodebuild ... 2>&1 | grep -E "error:|warning:")
  - Fix compilation errors in the relevant Swift files (no logic changes — only syntax/import/framework issues)
  - Re-run until BUILD SUCCEEDED
  - Do NOT change any HeuristicParams.swift values as fixes — compilation fixes only

Common issues to watch for:
  - Missing framework imports (Vision, ARKit, AVFoundation, CryptoKit) — add import statements
  - `simd_float4x4` usage: import simd or use `SIMD4<Float>` equivalents
  - `CVPixelBuffer` locking: CVPixelBufferLockFlags(rawValue: 0)
  - ARRaycastQuery initializer signature changed between iOS versions — check Apple docs for iOS 17 variant

Step 2: Once build succeeds, confirm HeuristicParams.swift hash compiles correctly.

  <what-built>
Task 1 confirmed the harness builds. Now the harness needs to be installed on a real iOS 17+ iPhone and run through a smoke test to confirm the full pipeline works: camera open -> front shot -> pose detection -> side shot -> measurement computation -> JSONL written to disk.

The simulator cannot run ARKit world tracking (required for scale calibration). A physical device is mandatory.
  </what-built>
  <how-to-verify>
1. Connect your iPhone (iOS 17+, preferably non-Pro / non-LiDAR — the majority install base) to your Mac.

2. In Xcode:
   - Open ~/sikai-spike-harness/SikAISpike.xcodeproj
   - Select your iPhone as the run target
   - Add your Apple Developer account (Xcode -> Settings -> Accounts) if not already added
   - Use a Personal Team provisioning profile (free tier is fine for personal device install)
   - Build and run (Cmd+R)

3. On the iPhone, run through a smoke-test scan with yourself as subject:
   - Set Tester to "T1" and Condition to "assisted_studio"
   - Stand approximately 2-3 meters from the phone (prop it against something at chest height)
   - Tap "Capture Front Shot" — the app should request camera permission if first run; grant it
   - Confirm that a front photo is captured (you should see a confirmation on screen)
   - Turn 90 degrees to your right (now standing sideways to the phone)
   - Tap "Capture Side Shot"
   - Tap "Process & Save"
   - Confirm that circumference values appear on screen (even if the numbers look wrong — this is just a smoke test, not real data)
   - Tap the Export button and confirm scans.jsonl can be shared via AirDrop or Files

4. Use Files app (or AirDrop the file to your Mac) to verify the scans.jsonl contents:
   - Open the file and confirm it contains at least one line of valid JSON
   - Confirm the JSON line contains: tester_id, session_id, scan_id, timestamp, computed_circumferences_cm, heuristic_params_hash
   - Confirm heuristic_params_hash is a 64-character hex string (not "hash-error")
   - Confirm computed_circumferences_cm.waist, .hip, and .chest are all non-null numbers

5. Verify ARKit scale source:
   - Check the scale_source field in the JSONL row: it should say "arkit" if ARKit plane detection worked, or "userHeight" if fallback was used
   - If scale_source is "none" for every scan, there is a bug in ScaleService.swift — fix before data collection

If the smoke test produces a valid JSONL row, data collection can proceed.
  </how-to-verify>
